# mainapp-leyu-resource

## 项目简介

`mainapp-leyu-resource` 是一个用于归档和发布多个独立 HTML 页面的资源仓库。本项目不针对任何特定域名或网站，仅作为静态 HTML 页面的集中存储与分发用途。仓库中的页面均为独立开发、独立维护，彼此之间无强关联。

## 目录结构

```
mainapp-leyu-resource/
├── pages/          # 存放各个独立 HTML 页面及其相关资源
├── assets/         # 公共静态资源（CSS、JS、图片等）
├── index.html      # 仓库首页导航（可选）
└── README.md       # 本文件
```

- `pages/`：每个子目录代表一个独立页面，包含其 HTML、样式、脚本及依赖文件。
- `assets/`：存放多个页面共用的资源文件，避免重复引用。
- `index.html`：如存在，可作为仓库内页面的简易导航入口。

## 页面归档说明

本仓库中的 HTML 页面以独立单元形式归档，每个页面拥有完整的结构与功能。页面之间无数据共享或依赖关系，可直接通过浏览器打开运行。

- 页面命名遵循 `pages/页面名称/index.html` 的约定。
- 每个页面目录内可包含其专属的 CSS、JavaScript 及媒体文件。
- 部分页面可能需要通过本地服务器运行（如涉及跨域请求或模块加载），建议使用 `http-server` 或 `live-server` 等工具。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录，放入 `index.html` 及相关资源，并在 `index.html`（如有）中添加导航链接。
- 更新页面：直接修改对应目录下的文件，注意保持与其他页面的独立性。
- 删除页面：移除对应目录及 `index.html` 中的导航链接。
- 公共资源：如新增公共资源，请放入 `assets/` 并遵循已有文件结构。

## 使用方式

1. 克隆本仓库：
   ```bash
   git clone https://github.com/your-username/mainapp-leyu-resource.git
   ```
2. 进入目录，直接打开 `pages/` 下任意 `index.html` 文件即可查看对应页面。
3. 如需整体浏览，可打开仓库根目录下的 `index.html`（若存在）。

## 注意事项

- 本仓库仅用于 HTML 页面的归档与发布，不涉及任何服务端逻辑或用户数据收集。
- 页面内容可能随时间更新，请以仓库最新提交为准。
- 如遇到页面无法正常显示，请检查文件路径是否正确或是否存在跨域限制。
